# Тесты медицинских алгоритмов

Эта папка содержит автоматические тесты для медицинских расчетов.

Тесты не используют базу данных и не создают записей.

## Запуск

```cmd
pytest
```

## Проверяемые блоки

Тесты проверяют папку:

```text
app/medical_algorithms/
common.py             — перевод значений, возраст, пол
ckd_stage.py          — стадии ХБП и нормализация русской С
egfr.py               — CKD-EPI 2021 creatinine
cockcroft_gault.py    — Cockcroft–Gault
albuminuria.py        — ACR и A1/A2/A3
prognosis.py          — KDIGO-прогноз
metrics.py            — объединенный расчет показателей
calculations.py       — переходный мостик совместимости
```

и переходный мостик:

```text
app/calculations.py
```

## Установка pytest

Из корня проекта:

```cmd
pip install -r requirements-dev.txt
```

## Запуск тестов

Из корня проекта:

```cmd
pytest
```

Более подробный вывод:

```cmd
pytest -v
```

Запуск одного файла:

```cmd
pytest tests\test_ckd_stage.py -v
```

## Что должно быть в норме

Если все хорошо, будет примерно:

```text
tests passed
```

или список тестов с `PASSED`.

## Если тест упал

Это не всегда значит, что “сломалось приложение”. Это значит, что конкретная расчетная функция стала выдавать другой результат.

После этого надо смотреть:

1. что изменилось в функции;
2. корректна ли сама формула;
3. не поменялось ли округление;
4. не изменилась ли единица измерения.

## Важное ограничение

Эти тесты не заменяют медицинскую валидацию.

Они нужны, чтобы зафиксировать поведение кода и быстро видеть, что мы случайно не сломали:

- CKD-EPI 2021;
- Cockcroft–Gault;
- стадии ХБП С1/С2/С3а/С3б/С4/С5;
- русскую букву `С`;
- ACR и A1/A2/A3;
- прогноз KDIGO;
- общий расчет показателей;
- мостик `app/calculations.py`.
