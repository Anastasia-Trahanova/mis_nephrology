# Исправление live-проверки дат v2

Назначение архива: исправить ошибку в frontend-проверке дат, из-за которой нормальная дата приёма могла подсвечиваться как «Некорректная дата».

В версии v2 исправлен сам скрипт применения патча: теперь замена JavaScript-функции выполняется безопасно через `lambda`, поэтому регулярные выражения внутри JS не ломают Python `re.sub`.

## Как применить

Распаковать архив в корень проекта с заменой файлов.

Запустить:

```cmd
python scripts\apply_date_validation_timezone_fix.py
```

Ожидаемый результат:

```text
OK: parseIsoDate исправлен без UTC-сдвига дат.
```

## Проверка

```cmd
pytest tests/layer/test_simple_form_guard_date_parser.py
pytest tests/layer/test_visit_date_rule_combinations.py
pytest tests/layer
```

## Ручная проверка

1. Открыть новый повторный приём.
2. Поставить дату приёма раньше сегодняшней, например 03.07.2026 при текущей дате 05.07.2026.
3. Поле «Дата приёма» не должно подсвечиваться как «Некорректная дата».
4. Дата приёма раньше даты рождения всё ещё должна давать ошибку.
5. Дата следующего визита раньше даты приёма или раньше текущей даты всё ещё должна давать ошибку.
