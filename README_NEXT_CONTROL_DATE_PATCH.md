# NEXT CONTROL DATE PATCH

## Назначение

Этот патч убирает автоподстановку старой даты следующего контроля в форму нового приёма.

Правильная логика:

- дата последнего приёма отображается только как статичный текст сверху формы;
- дата нового приёма — отдельное поле нового приёма;
- дата следующего контроля относится к новому приёму и по умолчанию должна быть пустой;
- если врач сам заполнил дату следующего контроля, тогда она проверяется;
- дата приёма может быть раньше текущей даты, потому что врач может вносить приём задним числом.

## Файлы

```text
scripts/apply_next_control_date_patch.py
```

Точечно правит `app/templates/appointment_form/_conclusion.html`:
находит `<input name="next_control_date">`, удаляет автоподстановку старого значения и оставляет `value=""`.

```text
tests/layer/test_visit_date_rule_combinations.py
```

Проверяет 15+ комбинаций дат:

- дата приёма в прошлом допустима;
- дата приёма раньше даты рождения запрещена;
- дата рождения в будущем запрещена;
- пустая дата следующего контроля допустима;
- дата следующего контроля раньше даты приёма запрещена;
- дата следующего контроля раньше текущей даты запрещена;
- даты исследований позже даты приёма запрещены;
- даты исследований до/в день приёма допустимы;
- некорректные форматы дат не роняют систему.

## Как применить

Распаковать архив в корень проекта.

Потом выполнить:

```cmd
python scripts\apply_next_control_date_patch.py
```

Проверить:

```cmd
pytest tests/layer/test_visit_date_rule_combinations.py
pytest tests/layer
```

## Ручная проверка

1. Открыть новый приём существующего пациента.
2. Убедиться, что сверху есть фраза: `В форму подгружены данные последнего приёма (...)`.
3. Убедиться, что поле `Дата следующего контроля` пустое.
4. Поставить дату приёма 03.07.2026 при текущей дате 05.07.2026 — ошибки быть не должно.
5. Поставить дату следующего контроля 02.07.2026 — должна появиться ошибка.
6. Поставить дату следующего контроля 06.07.2026 — ошибки быть не должно.
