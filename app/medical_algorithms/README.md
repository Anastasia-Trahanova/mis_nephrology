# Папка `medical_algorithms`

Эта папка содержит медицинские расчетные алгоритмы проекта МИС ХБП/нефрологии.

## Зачем вынесли алгоритмы отдельно

Раньше медицинские расчеты лежали в общем файле `app/calculations.py` и использовались вперемешку с кодом приложения.

Теперь логика постепенно разделяется:

```text
app/medical_algorithms/  — медицинские формулы и категории
app/database.py          — работа с PostgreSQL
app/routers/             — страницы и обработка запросов FastAPI
app/calculations.py      — временный мостик совместимости для старого кода
```

## Главное правило этой папки

Функции внутри `medical_algorithms` должны быть “чистыми”:

```text
получили значения на вход
посчитали результат
вернули результат
```

Они не должны:

```text
ходить в базу данных
читать .env
работать с HTML
работать с request/session
делать INSERT/UPDATE
знать про FastAPI
```

Это нужно, чтобы медицинские расчеты можно было отдельно тестировать, документировать и в будущем обосновывать при регистрации медицинского модуля.

## Файлы

```text
common.py             — общие технические функции: перевод в число, расчет возраста, определение пола
egfr.py               — расчет eGFR по CKD-EPI 2021 creatinine
cockcroft_gault.py    — расчет клиренса креатинина по Cockcroft–Gault
ckd_stage.py          — определение и нормализация категории СКФ: С1, С2, С3а, С3б, С4, С5
albuminuria.py        — расчет ACR и категории альбуминурии A1/A2/A3
prognosis.py          — прогноз ХБП по матрице KDIGO: категория СКФ + альбуминурия
metrics.py            — объединенный расчет основных показателей для приема
__init__.py           — общий экспорт функций из папки
```

## Единицы измерения

### Креатинин крови

Входное значение:

```text
мкмоль/л
```

Используется в:

```text
egfr.py
cockcroft_gault.py
metrics.py
```

### eGFR CKD-EPI

Выходное значение:

```text
мл/мин/1,73 м²
```

### Cockcroft–Gault

Выходное значение:

```text
мл/мин
```

### Альбумин мочи

Поддерживаемые входные единицы:

```text
mg_l — мг/л
g_l  — г/л
```

### Креатинин мочи

Поддерживаемые входные единицы:

```text
mmol_l — ммоль/л
umol_l — мкмоль/л
```

### ACR

Основной выход:

```text
мг/ммоль
```

Дополнительный выход для отображения:

```text
мг/г
```

## Категории СКФ

В проекте используется русская буква `С`, не латинская `C` и не `G`.

```text
С1  — eGFR ≥ 90
С2  — eGFR 60–89
С3а — eGFR 45–59
С3б — eGFR 30–44
С4  — eGFR 15–29
С5  — eGFR < 15
```

## Категории альбуминурии

```text
A1 — ACR < 3 мг/ммоль
A2 — ACR 3–30 мг/ммоль
A3 — ACR > 30 мг/ммоль
```

## Как это подключается к старому коду

Пока старый код может продолжать импортировать функции из:

```python
from app.calculations import calculate_all_metrics
```

`app/calculations.py` оставляем как мостик совместимости. Позже он будет просто переэкспортировать функции из этой папки.
