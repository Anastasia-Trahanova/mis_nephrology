# Патч страницы журнала работы МИС

Что входит:

- `app/routers/admin.py` — обновлённый роут админского журнала.
- `app/templates/admin/audit.html` — обновлённая общая страница журнала.

Что изменено:

1. В общей таблице журнала дата и время выводятся в две строки внутри одной ячейки.
2. Столбец `Пациент / приём` заменён на `Пациент`: выводится только пациент, без номера приёма.
3. Столбец `Переходы` удалён.
4. В столбце `Детали` выводится ссылка `Подробнее`:
   - для событий с `appointment_id` — на протокол аудита всего приёма `/admin/audit/appointment/{appointment_id}`;
   - для событий без `appointment_id` — на протокол одного события `/admin/audit/{event_id}`.
5. Над таблицей добавлены рабочие фильтры: категория, действие, результат, роль, пользователь, ID пациента, ID приёма, даты, лимит, только ошибки/отказы.
6. Добавлен включённый по умолчанию фильтр `скрыть просмотры журнала и протоколов аудита`, чтобы строки `open_admin_audit`, `open_admin_audit_event`, `open_admin_appointment_audit` не забивали общую страницу.
7. CSV-выгрузка использует те же фильтры, что и таблица.

Как применить:

1. Скопировать папку `app` из архива в корень репозитория с заменой файлов.
2. Запустить приложение.
3. Открыть `/admin/audit` под пользователем с ролью `admin`.
4. Проверить фильтры и ссылку `Подробнее` на событиях с приёмом.

Рекомендуемая проверка:

```bat
pytest tests/layer
```

Если менялся только интерфейс админского журнала, дополнительно стоит вручную открыть:

- `/admin/audit`
- `/admin/audit?hide_admin_audit_views=0`
- `/admin/audit?only_errors=1`
- `/admin/audit/appointment/<существующий_appointment_id>`
