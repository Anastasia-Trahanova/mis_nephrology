# Simple form validation patch

Назначение архива: добавить спокойный минимальный слой предупреждений формы без лишних подсказок для врача.

## Что внутри

```text
app/validation.py
app/templates/_clinical_validation.html
app/templates/appointment_form/_calculation_hints.html
app/static/js/simple_form_guard.js
scripts/apply_simple_form_ui_patch.py
tests/layer/test_simple_validation_messages.py
tests/layer/test_simple_form_guard_assets.py
```

## app/validation.py

Серверная проверка клинических полей.

Что изменено:

- все подробные сообщения с диапазонами заменены на короткое `Неверное значение`;
- `1015` для удельного веса продолжает приниматься и нормализоваться внутри приложения, но врач не видит сообщение об этом;
- `1500` для удельного веса даёт только `Неверное значение`;
- частично заполненная альбуминурия больше не блокирует сохранение;
- добавлены короткие проверки дат:
  - `Дата рождения не может быть в будущем`;
  - `Дата приёма не может быть раньше даты рождения`;
  - `Дата следующего визита не может быть раньше даты приёма`;
  - `Дата следующего визита не может быть раньше текущей даты`;
  - `Дата исследования не может быть позже даты приёма`.

Важно: функция `validate_appointment_form(...)` осталась совместимой со старым вызовом из двух аргументов.
Новые проверки даты рождения включаются, если передать `birth_date_value=...`.

## app/templates/_clinical_validation.html

Единый partial для коротких сообщений формы.

Что делает:

- показывает серверные ошибки, если они переданы в шаблон как `errors`;
- содержит скрытый блок для клиентского сообщения `Не все обязательные поля заполнены`;
- подключает `app/static/js/simple_form_guard.js`.

## app/static/js/simple_form_guard.js

Клиентская проверка перед отправкой формы.

Что делает:

- не даёт отправить форму, если не заполнены поля `required`;
- показывает `Не все обязательные поля заполнены`;
- прокручивает к первому незаполненному обязательному полю;
- проверяет даты на клиенте;
- не показывает диапазоны лабораторных значений;
- не показывает сообщение о нормализации удельного веса;
- не ругается на частично заполненную альбуминурию, если поля не отмечены как `required`.

## app/templates/appointment_form/_calculation_hints.html

Нейтральные короткие подсказки для расчётов.

Содержит два макроса:

```jinja2
{{ egfr_hint() }}
{{ acr_hint() }}
```

Они показывают под таблицами:

```text
Для расчёта СКФ заполните дату рождения, пол, креатинин крови, рост и вес.
Для расчёта ACR заполните альбумин мочи и креатинин мочи.
```

Это не предупреждения и не ошибки.

## scripts/apply_simple_form_ui_patch.py

Безопасный скрипт для точечных правок существующих шаблонов.

Он:

1. убирает лишнее сообщение о подгруженных данных в `new_appointment.html`;
2. оставляет одну фразу: `В форму подгружены данные последнего приёма (дата)`;
3. удаляет видимую фразу про код динамического расчёта ИМТ;
4. вставляет короткие подсказки под таблицами СКФ и альбуминурии.

Запуск из корня проекта:

```cmd
python scripts\apply_simple_form_ui_patch.py
```

Скрипт не перестраивает дизайн и не меняет расположение блоков.

## Как применять

### Вариант 1 — просто заменить файлы из архива

Можно заменить/добавить эти файлы:

```text
app/validation.py
app/templates/_clinical_validation.html
app/templates/appointment_form/_calculation_hints.html
app/static/js/simple_form_guard.js
tests/layer/test_simple_validation_messages.py
tests/layer/test_simple_form_guard_assets.py
```

После этого клиентская проверка обязательных полей заработает там, где уже подключён `_clinical_validation.html`.

### Вариант 2 — дополнительно применить точечные правки шаблонов

Чтобы автоматически убрать лишнее сообщение о подгрузке данных, удалить видимую фразу про BMI и вставить подсказки под таблицы:

```cmd
python scripts\apply_simple_form_ui_patch.py
```

## Что проверить

```cmd
pytest tests/layer/test_simple_validation_messages.py
pytest tests/layer/test_simple_form_guard_assets.py
pytest tests/layer
```

Ручная проверка в браузере:

1. Новый пациент: нажать сохранить без фамилии/имени/врача — должно появиться `Не все обязательные поля заполнены`, фокус уйдёт к первому пустому полю.
2. Новый пациент: поставить дату рождения завтра — должно появиться `Дата рождения не может быть в будущем`.
3. Повторный приём: поставить дату следующего визита раньше даты приёма — должно появиться `Дата следующего визита не может быть раньше даты приёма`.
4. Дата исследования позже даты приёма — должно появиться `Дата исследования не может быть позже даты приёма`.
5. Удельный вес `1015` — не должно быть никакого предупреждения.
6. Удельный вес `1500` — серверная проверка должна вернуть `Неверное значение`.
7. Заполнить только альбумин мочи без креатинина мочи — форма не должна ругаться сообщением про обязательные поля.
