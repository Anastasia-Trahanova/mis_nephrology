# База данных

Финальная минимальная структура SQL-файлов:

1. `01 создание таблиц.sql` — основные таблицы системы без тестовых данных.
2. `02 настройка связей ключей и ограничений.sql` — внешние ключи, CHECK, UNIQUE, индексы для основных таблиц.
3. `03 создание и заполнение таблиц МКБ.sql` — таблицы МКБ, связи, view `appointment_icd10_diagnoses_view` и наполнение справочника.
4. `04 заполнение тестовыми данными.sql` — демонстрационные данные. В production не запускать.
5. `05 оценка целостности.sql` — проверочные запросы качества данных.
6. `06 аналитические запросы.sql` — клинические/регистровые тестовые выборки.

Порядок запуска для чистой базы:

```sql
-- 01 создание таблиц.sql
-- 02 настройка связей ключей и ограничений.sql
-- 03 создание и заполнение таблиц МКБ.sql
-- 04 заполнение тестовыми данными.sql      -- только dev/demo
-- 05 оценка целостности.sql
-- 06 аналитические запросы.sql
```

Стандарт стадий ХБП в базе и коде: русская `С` — `С1`, `С2`, `С3а`, `С3б`, `С4`, `С5`.
